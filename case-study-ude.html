UDE 料號查詢
自動化平台

以 n8n 為核心打造企業內部料號查詢與資料整合平台，整合 MySQL、Excel 與即時查詢介面，將原本繁瑣的人工查詢 / 比對流程完全自動化。

n8n · MySQL · JavaScript 企業內部系統 查詢時間 −90%
01 — 專案背景

這是什麼系統？解決什麼業務問題？

UDE 是一家製造業企業，料號資料散落在多個 Excel 試算表與 MySQL 資料庫中。業務、倉儲、採購三個部門都需要查詢料號相關資訊（規格、庫存、供應商等），但每次查詢都需要人工開啟檔案、逐一比對，效率極低。

這個專案的目標是：把這些分散的資料來源整合起來，建立一個任何人都能立刻查的內部工具，不需要懂 SQL，不需要開 Excel，輸入料號就能看到所有相關資訊。

客戶原話：「我們兩個人要花半小時對一批料號，還常常對錯，能不能讓系統自己去找？」
02 — 問題與痛點

人工查詢流程的核心問題

痛點 1 料號資料橫跨多個 Excel 版本與 MySQL 資料表，欄位命名不一致，人工比對容易出錯。
痛點 2 每次查詢需要兩人協作：一人查 Excel、一人查資料庫，對完再人工填入 Google Sheet，耗時數分鐘到半小時不等。
痛點 3 資料版本不同步——Excel 更新了但資料庫沒更新，導致查詢結果矛盾，需要追溯哪份才是最新的。
痛點 4 現有流程完全依賴特定人員，一旦那個人不在，其他人根本不知道去哪裡找資料。
03 — 解法與技術架構

以 n8n 為核心的整合架構

選擇 n8n 作為核心引擎，原因是它能以視覺化 Workflow 串接資料庫、HTTP API 與前端，部署在企業內網後不依賴外部服務，也容易讓非技術人員理解流程。

04 — 我的負責範圍

從需求訪談到系統上線，全程一人主導

  • 1與業務、倉儲部門訪談，梳理實際查詢流程與痛點，整理出系統需求文件
  • 2設計 MySQL schema——包含料號主表、欄位正規化規則與 JOIN key 邏輯
  • 3建立 n8n Workflow：Excel 定時匯入、查詢 API 接口、資料異常通知
  • 4開發前端查詢介面（HTML + DataTables），支援即時搜尋、欄位篩選與匯出
  • 5效能調優——針對大料號資料量優化 MySQL index 與 n8n 節點參數
  • 6系統部署至企業內網，撰寫操作說明文件，培訓使用者上手
05 — 實作重點與難點

三個讓這個案子不容易做的技術挑戰

挑戰 A 欄位命名不一致：不同部門的 Excel 同一個欄位叫不同名字（「料號」vs「Part No.」vs「PN」），需要建立對應表並在匯入時自動標準化。
挑戰 B 大資料量效能：料號總筆數超過萬筆，前端即時搜尋在無 index 的情況下明顯延遲。透過 MySQL index 優化與分頁機制，搜尋反應時間壓到 200ms 以下。
挑戰 C 讓非技術人員能維護：客戶希望未來自己能更新資料，所以系統設計成「上傳 Excel 就自動同步」，而不是要求他們操作資料庫。
