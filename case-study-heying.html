鶴英｜品牌官網設計

為鶴英品牌從零打造具視覺識別的官方網站，涵蓋品牌形象呈現、RWD 響應式設計與清楚的行動呼籲，讓潛在客戶能快速找到品牌並產生轉換。

HTML · CSS · JavaScript 品牌官網 / 商業網站 RWD 全裝置適配
01 — 專案背景

品牌需要的第一個對外窗口

鶴英是一個具有獨特定位的品牌，但在接到這個專案時，他們完全沒有任何線上存在感——沒有官網、沒有品牌頁面，潛在客戶就算想找也找不到。

這個專案的挑戰不只是「做一個網站」，而是從零開始理解品牌的視覺語言，並把它轉化成一個能夠自我介紹的網站——讓第一次看到的人就能理解品牌是什麼、為什麼值得信任。

專案核心：網站不只是展示用，它要能讓潛在客戶「看完就想聯絡」，並且在各種裝置上都能正常閱讀，包括手機。
02 — 問題與痛點

沒有官網的品牌在市場上幾乎是隱形的

痛點 1完全沒有對外展示管道，潛在客戶透過搜尋或社群找到品牌名後，沒有任何可以深入了解的地方。
痛點 2沒有聯絡入口，有興趣的人不知道要怎麼洽詢，商機直接流失。
痛點 3缺乏一致的品牌視覺呈現，讓品牌看起來不夠專業，難以建立信任感。
03 — 解法與設計思路

從品牌定位出發，設計而非套模板

設計前先做品牌理解：鶴英代表的是什麼？目標客群是誰？視覺上要給人什麼感覺？把這些問題確認清楚之後，才開始動手設計。

整個網站設計圍繞三個核心原則：視覺識別一致、導覽清楚直覺、行動呼籲明確。每個頁面的設計決策都要能回答「這樣設計對品牌有什麼幫助？」

04 — 我的負責範圍

從品牌理解到上線部署，一人全程負責

  • 1品牌訪談與需求釐清——了解鶴英的定位、目標客群與對網站的期待
  • 2規劃網站頁面架構與資訊層次，決定呈現順序與視覺重點
  • 3設計品牌視覺系統——色彩、字型、間距，建立一致的視覺語言
  • 4以純 HTML / CSS / JS 手工切版，不依賴框架，確保輕量與速度
  • 5RWD 測試——桌面、平板、手機三個斷點全面確認視覺與操作體驗
  • 6完成後部署上線，交付可維護的程式碼結構與基本說明文件
05 — 設計重點

讓網站真正服務品牌的三個關鍵決策

設計重點不套模板，從品牌出發：市面上的網站模板很多，但鶴英有自己的視覺語言。從色調、字型選擇到版面比例，每個決策都有對應的品牌理由，不是隨機選的。
設計重點行動呼籲的層次：每個區塊都有明確的下一步引導——了解品牌 → 查看服務 → 聯絡我們。網站的結構設計讓訪客自然地走向轉換。
設計重點手機版的閱讀體驗：大多數訪客會從手機進入，所以手機版不是「縮小版桌面」，而是重新思考資訊在小螢幕上的最佳呈現方式。
