自動追更通報系統｜n8n 內容擷取自動化

我用 n8n 打造的第二個自動化工具。原本需要員工每天手動巡站確認小說與歌詞更新、複製內容回傳老闆；現在系統自動爬取最新資訊、擷取重點段落並自動寄信——全程零人工。

n8n · Web Scraping 每日排程全自動 零人工巡站
01 — 專案背景

每天重複的「巡站回報」，值得被自動化

這是我用 n8n 完成的第二個自動化案子。老闆有固定追蹤的小說與歌詞網站，原本的做法是：員工每天打開網站確認有沒有更新，有更新就把最新一集的內容複製下來，整理後回傳給老闆。

這件事每天都要做、內容高度重複、又不能漏——正是最適合自動化的工作型態。目標是讓整條「確認更新 → 擷取內容 → 回報老闆」的流程完全交給系統。

專案核心：不只是「抓到網頁」，而是抓對內容——系統要能判斷是否真的有新章節，並從整頁內容中擷取出老闆真正想看的重點段落。
02 — 問題與痛點

簡單的事每天做，累積起來就不簡單

痛點 1員工每天要手動打開多個網站逐一確認更新，重複、瑣碎、佔用工作時間。
痛點 2人工複製貼上容易漏抓、抓錯段落，回報品質不穩定。
痛點 3更新時間不固定，人工確認有時差——老闆想第一時間看到，人工流程做不到。
03 — 解法與流程設計

n8n 排程觸發，從爬取到寄信一條龍

整條流程建在 n8n 上：排程節點定時觸發，爬取目標網站的最新章節與歌詞頁面；接著比對紀錄判斷是否為新內容，是新內容才往下走。

確認有更新後，系統自動擷取重點段落、整理成易讀的信件格式，寄送到老闆信箱。整條流程視覺化呈現在 n8n 畫布上，維護與擴充都一目瞭然。

04 — 我的負責範圍

從需求釐清到穩定運行，全程一人完成

  • 1與需求方確認追蹤標的、更新頻率與希望收到的內容格式
  • 2分析目標網站結構，設計穩定的爬取與內容解析方式
  • 3在 n8n 建立排程觸發 → 爬取 → 更新判斷 → 內容擷取 → 寄信的完整工作流
  • 4設計重點段落擷取邏輯，把整頁內容濃縮成老闆要看的部分
  • 5設定異常處理——網站改版或爬取失敗時能被察覺，不會靜默漏報
  • 6上線後觀察運行狀況並調校，確保長期穩定
05 — 實作重點與難點

把「自動化」做到可靠的三個關鍵

實作重點更新判斷：爬得到網頁不等於抓得到「新內容」。系統會比對歷史紀錄，只有真正的新章節才觸發後續流程，避免重複寄信洗版。
實作重點重點擷取：整頁內容直接寄出沒人想看。擷取邏輯把導覽、廣告等雜訊排除，只留下核心段落，讓信件打開就是重點。
實作重點可維護性：流程全部視覺化在 n8n 上，之後要新增追蹤網站或改寄送對象，擴充節點即可，不用重寫程式。
